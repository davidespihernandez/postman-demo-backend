# Postman Demo Backend

Simple Node.js + Express API for testing Postman Git integration.

Data is stored in memory (no database), so restarting the server resets all customers.

## Requirements

- Node.js 18+ (or newer)
- npm

## Install

```bash
npm install
```

## Start

```bash
npm start
```

Server runs at `http://localhost:3000` by default.

Optional: run with watch mode for local development:

```bash
npm run dev
```

## Stop

In the terminal where the server is running, press:

```bash
Ctrl + C
```

## Customer Model

```json
{
  "id": "uuid",
  "name": "string",
  "email": "string",
  "phone": "string"
}
```

## Endpoints

### GET /customers

Returns all customers.

Example:

```bash
curl http://localhost:3000/customers
```

### GET /customers/{id}

Returns a single customer by ID.

Example:

```bash
curl http://localhost:3000/customers/3a199f49-f6a5-44da-890f-00e8fb9631b8
```

### POST /customers

Creates a customer. Body fields required: `name`, `email`, `phone`.

Example:

```bash
curl -X POST http://localhost:3000/customers \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Doe",
    "email": "jane@example.com",
    "phone": "+1-555-0100"
  }'
```

### PATCH /customers/{id}

Partially updates one or more fields: `name`, `email`, `phone`.

Example:

```bash
curl -X PATCH http://localhost:3000/customers/3a199f49-f6a5-44da-890f-00e8fb9631b8 \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "+1-555-0199"
  }'
```

## Notes

- IDs are UUIDs generated by the server.
- Data is not persisted; it lives in memory only.
